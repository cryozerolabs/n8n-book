# 错误处理

> 📌 **本节目标**：让工作流更稳定可靠

## 为什么要处理错误

工作流在生产环境运行，难免遇到：

- 网络超时
- API 限流
- 数据格式异常

## n8n 错误处理机制

### 1. 节点级重试

在节点设置中开启：

- **重试次数**：失败后重试几次
- **重试间隔**：每次重试等待时间

### 2. Error Trigger

专门处理错误的触发器：

```
[任意工作流出错] → [Error Trigger] → [发送告警]
```

### 3. 条件分支

用 IF 节点判断是否成功：

```
[HTTP Request] → [IF 状态码=200] → 成功处理
                                  → 失败处理
```

## 最佳实践

1. **关键节点开启重试**
2. **配置全局错误告警**
3. **记录错误日志**
